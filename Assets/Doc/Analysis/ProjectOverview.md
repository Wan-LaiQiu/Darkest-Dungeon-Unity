# 项目全量分析总结 (Project Overview)

## 1. 项目简介
本项目是使用 **Unity 2019/2020+** 开发的《暗黑地牢》(Darkest Dungeon) 的高度还原复刻版本。它旨在复现原版游戏的核心战斗机制、地牢探索流程、庄园管理系统以及独特的艺术风格和氛围。

### 核心特性
- **高度还原的战斗系统**：包括排位机制、先攻判定、命中/闪避算法、压力系统、美德/折磨判定、死亡之门等。
- **随机地牢生成**：基于图论的房间与走廊连接生成算法，支持多种地牢环境（遗迹、荒野、贫民窟、海湾）。
- **庄园管理**：完整的城镇建设系统，包含铁匠铺、行会、驿站马车、疗养院等功能建筑。
- **技能与 Buff 系统**：支持复杂的触发规则（光照、排位、血量等）和多种持续时间类型。
- **数据驱动设计**：几乎所有的游戏数值、英雄属性、怪物配置和任务定义都通过外部 JSON 文件管理。

---

## 2. 技术栈
- **引擎**: Unity 3D
- **语言**: C#
- **渲染**: 内置渲染管线 (Built-in RP) / 自定义着色器 (针对 2D 骨骼动画和特效)
- **动画**: Spine (Esoteric Software) 2D 骨骼动画
- **音频**: FMOD Studio 集成
- **数据存储**: JSON (Newtonsoft.Json) + 自定义二进制序列化 (存档)
- **网络**: Photon Unity Networking (PUN) (项目中包含网络代码，可能用于同步或多人模式扩展)

## 4. 关键系统概览

### 4.1 核心驱动 (Core Driver)
项目由 `DarkestDungeonManager` 单例驱动，它负责初始化 `DarkestDatabase` 并持有 `Campaign` (当前战役) 的引用。

### 4.2 战斗引擎 (Combat Engine)
战斗由 `Round` 类管理回合流转，`BattleSolver` 处理具体的伤害和概率计算。
- **排位机制**: 4x4 的阵型，技能的使用和目标选择严格受排位限制。
- **压力系统**: 独特的心理状态模拟，影响战斗表现和长期生存。

### 4.3 探索流程 (Exploration Flow)
玩家在房间和走廊之间移动。
- **光照**: 随着探索消耗火把，影响难度和奖励。
- **随机性**: 陷阱、奇物、饥饿判定和侦察机制增加了策略深度。

### 4.4 数据驱动 (Data Driven)
所有实体（英雄、怪物、物品、技能）的定义都解耦到 JSON 中，使得数值平衡和内容扩展非常方便。

---

## 5. 开发建议与扩展
- **数值调整**: 大部分平衡参数位于 `Resources/Data/` 下的 JSON 文件中。
- **新增英雄/怪物**: 需要对应的 Spine 动画预制体并在 JSON 数据库中注册其职业和技能数据。
- **存档维护**: 存档采用 `SaveCampaignData` 结构，通过 `SaveLoadManager` 进行持久化。
